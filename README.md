# AstrBot Plugin: TodoPal

AI 驱动的智能待办事项管理插件。只需通过自然语言描述，即可自动解析时间、日期和任务内容。

## 功能特性

- **自然语言解析**：直接用平时说话的方式描述待办事项，无需复杂格式。
- **智能时间识别**：自动理解“明天下午”、“下周三”、“晚上8点”等时间表达。
- **多任务处理**：一条消息中可以包含多个待办事项。

## 使用方法

本插件使用特定的前缀来区分不同的操作意图，支持以下三个核心指令：

### 1. `todo`：新建/覆盖待办清单
使用 `todo` 开头，表示声明一个**全新**的待办清单。如果当天已有记录，使用该命令并在确认后，会**直接覆盖**原有记录。

**示例：**
> 用户：`todo 明天下午3点给谭老师发邮件，晚上8点健身`
> 
> Bot：
> 【新建/覆盖模式】
> 2026年3月17日 星期二
> 
> 1. 15:00 给谭老师发邮件
> 2. 20:00 健身
> 
> 如果正确，请回复“确认”。

### 2. `add`：追加待办事项
使用 `add` 开头，表示在现有的待办清单上**追加**新的任务，不会影响已有的记录。

**示例：**
> 用户：`add 今天下午再加个会`
> 
> Bot：
> 【追加模式】
> 2026年3月16日 星期一
> 
> 1. 下午再加个会
> 
> 如果正确，请回复“确认”。

### 3. `done`：标记完成
使用 `done` 开头，快速将某一项或多项任务标记为已完成（✅）。支持通过序号或内容片段进行匹配。

**示例 1：按序号完成（支持批量）**
> 用户：`done 1, 2` 或 `done 第1条`
> 
> Bot：
> 太棒了！已更新状态：
> 2026年3月16日 星期一
> 
> 1. ✅ 15:00 给谭老师发邮件
> 2. ✅ 20:00 健身

**示例 2：按内容模糊匹配**
> 用户：`done 健身`
> 
> Bot：
> 太棒了！已更新状态：... (对应项目打上✅)

## 数据存储

本插件使用本地 JSON 文件存储数据，无需额外数据库配置。

**文件结构**：
```text
data/plugin_data/todopal/
├── {platform}/           # 平台名称 (如 qq, wechat)
│   └── {user_id}/        # 用户 ID
│       └── {year}/       # 年份
│           └── {month}/  # 月份
│               └── {date}.json  # 每天一个文件
```

**数据格式示例**：
```json
[
  {
    "id": "20260316-abcdef",
    "date": "2026-03-16",
    "time": "15:00",
    "content": "给谭老师发邮件",
    "status": "pending",  // pending: 待办, done: 已完成
    "created_at": "2026-03-16 10:00:00",
    "updated_at": "2026-03-16 10:00:00",
    "done_at": null,
    "source_text": "todo 明天下午3点给谭老师发邮件"
  }
]
```

## 安装

本插件专为 [AstrBot](https://github.com/AstrBotDevs/AstrBot) 设计。请将其放置在 AstrBot 的插件目录中。
