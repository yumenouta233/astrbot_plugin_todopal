# Changelog

## [1.12.7] - 2026-03-22

### Changed
- 新增同日任务自检合并：读取与保存时自动按 `content+time+tag` 合并重复项，兼容顺延残留或其他来源的重复数据。
- 合并状态规则优化：同组任务状态按 `done > rolled_over > pending` 归并，避免同任务状态分裂。
- 自检落盘生效：发现重复后会自动写回清理，后续 `check 原始` 不再持续出现同内容双条。

## [1.12.6] - 2026-03-22

### Changed
- 顺延机制从“复制”调整为“剪切迁移”：前一天未完成任务会直接迁移到次日并标记 `↪`，不再保留前一天同一条未完成副本。
- 顺延兼容清理增强：当次日已存在由旧机制复制出的顺延项时，会自动清理前一日对应的重复源任务，避免历史双份残留。
- 顺延去重判定覆盖 `done` 状态：若次日已有同内容任务（含已完成），不会再次迁移生成重复项。

## [1.12.5] - 2026-03-22

### Changed
- `check 今天` 的安排表新增同内容合并展示：当日任务与顺延任务同内容时仅显示一条，避免重复刷屏。
- 合并项支持“含顺延”标识，并增加“已合并重复项 N 条”提示，便于理解展示与原始数据差异。
- 底部提示文案优化：明确 `check 原始` 用于查看完整明细和精确序号操作。

## [1.12.4] - 2026-03-22

### Changed
- 修复删除命令日期识别：`del 明天1` 现在会删除明天清单中的第 1 项，不再默认删今天。
- 安排表展示补充顺延标识：顺延任务在 `check 今天` 中显示 `↪`，便于区分来源。
- 新增新增待办日期护栏：用户输入中无明确日期表达时，解析结果强制回落到当天，降低 typo 触发误日期风险。
- 顺延去重增强：按 `content+time+tag` 去重，避免顺延任务与当天同内容任务重复堆叠。

## [1.12.3] - 2026-03-21

### Changed
- 安排表中的序号改为沿用原始清单序号，避免 `check 今天` 与 `done N` 的序号语义不一致。
- 新增撤销完成能力：支持 `undone N`、`undo N`、`撤销完成 N`、`取消完成 N`、`取消done N`。
- 新增 `todo_undone` 工具接口，便于 Agent 在误完成后直接回滚状态。

## [1.12.2] - 2026-03-21

### Changed
- 优先任务队列排序规则调整为“先按标签类别，再按优先级”，同标签内再按轻重缓急排序。
- 优先任务队列展示新增标签分组标题，移动端阅读时更容易按类别执行。

## [1.12.1] - 2026-03-21

### Changed
- 安排表展示重构：无固定时间任务默认仅按轻重缓急输出“优先任务队列”，不再显示内部插空时段，避免用户误解为硬性预约时间。
- 安排表新增分区展示：`固定时段任务` 与 `优先任务队列`，候补区支持优先级等级展示。
- 新增配置项：`todo_plan_show_virtual_time`（默认关闭）与 `todo_plan_show_priority_level`（默认开启）。

## [1.12.0] - 2026-03-21

### Added
- 新增配置页标签 emoji 映射：`todo_tag_meta`，支持 `标签|emoji` 格式用于移动端展示。
- 新增安排表相关配置：`todo_check_default_mode`、`todo_llm_priority_enable`、`todo_workday_start/end`、`todo_fixed_task_block_minutes`、`todo_default_flexible_duration_minutes`、`todo_min_gap_minutes`。

### Changed
- `check` 默认视图可配置为安排表；支持 `check 原始` 显示旧版原始清单。
- 安排表逻辑支持固定时间任务优先、无固定时间任务按优先级插空，并输出候补区。
- 待办标签展示由 `[标签]` 升级为 emoji/文本前缀，优化手机端阅读体验。

## [1.11.1] - 2026-03-21

### Changed
- 配置页标签能力增强：新增 `todo_use_config_tags_only` 与 `todo_allow_tag_command_edit`，可切换“配置页全局标签优先”策略。
- 标签编排提示新增“标签来源”提示，开启全局标签模式时明确显示来自配置页。
- 开启全局标签模式后，`标签新增/删除/改名` 命令会被禁用并提示到配置页面维护。

## [1.11.0] - 2026-03-21

### Added
- 新增标签管理命令：`标签列表/标签新增/标签删除/标签改名`，支持用户侧标签增删改查。
- 新增批量新增后的标签编排确认：支持 `1x3` 或 `1,x,3`，数字绑定标签、`x` 丢弃条目、`0` 保留但不打标签。

### Changed
- 待办确认流程升级为“标签编排后保存”，仍保留“确认/取消”快捷路径。
- 待办展示与工具文本输出支持标签显示，已保存条目写入 `tag_id/tag_name` 字段。

## [1.10.7] - 2026-03-21

### Changed
- 将提醒与诊断中的“未完成”口径统一为 `pending + rolled_over`，顺延任务也会触发提醒与计入未完成统计。
- 清单展示新增顺延标识：`rolled_over` 项显示 `↪`，便于区分当日任务与顺延任务。

## [1.4.0] - 2026-03-16

### Added
- **智能意图识别 (Smart Agent Mode)**：升级 `todo` 指令为智能入口。现在你可以直接说 `todo 把第二条改成开会` 或 `todo 买菜做完了`，插件会自动识别并执行 `fix` 或 `done` 操作，无需记忆特定指令前缀。
- 支持 `todo` 后接纯自然语言，如 `todo 看看还有什么` (Check) 或 `todo 加上买牛奶` (Add)。

### Changed
- `todo` 指令不再默认覆盖当日清单，而是默认为**智能追加/处理**模式。为了防止数据误删，现在 `todo` 和 `add` 都会追加待办事项。如果需要覆盖，请手动删除旧数据（覆盖功能将在后续版本重新设计）。
- 保持 `add`, `done`, `fix`, `check` 作为快捷指令，供需要精准控制或不希望消耗 LLM Token 的场景使用。

## [1.3.0] - 2026-03-16

### Added
- 新增 `fix` 指令：支持修改已存在的待办内容（如 `fix 2 改成开会`），解决了之前无法修改只能覆盖的问题。
- 新增 `check` 指令：支持随时查看今日待办清单，无需触发解析流程。

### Changed
- 优化 `todo` 指令逻辑：现在作为“重置/新建”指令，如果检测到当日已有数据，会发出覆盖警告，防止误操作。

### Fixed
- 修复 `done`、`check`、`fix` 指令执行时可能出现的 `async_generator` 报错或无响应问题（重构了内部处理函数的返回机制）。

## [1.2.0] - 2026-03-16

### Changed
- 重构命令路由逻辑，引入明确的前缀机制：
  - `todo`：新建并覆盖当前清单。
  - `add`：在当前清单基础上追加新事项。
  - `done`：标记任务完成（支持批量，如 `done 1, 2` 或 `done 开会`）。
- 取消了原来依赖 LLM 意图识别的“追加/覆盖”提示，改为由指令前缀显式控制。
- 优化了待办事项的展示格式，已完成的项现在会直观地显示 ✅ 标记。
- 移除了 `done` 指令对 LLM 的依赖，直接在本地进行正则和内容匹配，大幅提升响应速度和准确率。

## [1.1.0] - 2026-03-16

### Added
- 实现本地 JSON 存储功能，支持按用户、日期归档待办事项。
- 新增待办事项交互确认流程（Wait-for-Confirmation）。
- 支持“追加”或“覆盖”今日已存在的待办事项。
- 新增自然语言标记完成功能（支持“第X个做完了”或“买菜做完了”）。

### Fixed
- 修复 `AiocqhttpMessageEvent` 等部分适配器缺少 `platform_name` 属性导致的崩溃问题。
- 优化 LLM 解析逻辑，增强对非标准 JSON 返回的鲁棒性。

## [1.0.0] - 2026-03-16

### Added
- 初始版本发布。
- 支持 `todo` 关键词解析待办事项。
- 实现基于 LLM 的自然语言理解，自动提取日期、时间和任务内容。
- 支持一次性添加多个待办事项。
- 智能时间推断（默认解析为当天）。

### Fixed
- 修复 LLM 返回 Markdown 代码块格式导致解析失败的问题。
- 修复部分 AstrBot 版本下无法获取 Provider ID 的问题（兼容 v4.5.7+ 新 API）。
